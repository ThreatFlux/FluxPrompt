# Security Policy

## Supported Versions

FluxPrompt is under active development. Security fixes are prioritized for the latest published release and the `main` branch.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| `main` | Yes |
| Older releases | Best effort |

## Reporting a Vulnerability

Do not open a public GitHub issue for a potential vulnerability.

Preferred path:

1. Use GitHub's private vulnerability reporting flow for this repository if it is available.
2. If private reporting is unavailable, contact the repository maintainers privately before disclosing details publicly.
3. Include a clear description of impact, affected versions or commits, reproduction steps, and any mitigations you have already tested.

Please avoid posting proof-of-concept exploits, bypass details, or sensitive logs in public issues, pull requests, or discussions until a fix or mitigation is available.

## What To Include

- A concise summary of the issue and why it matters
- Affected versions, branches, or configuration conditions
- Reproduction steps or a proof of concept
- Expected impact and any known workarounds
- Relevant logs, traces, or sample inputs

## Response Expectations

- Initial acknowledgement within 3 business days
- A triage update within 7 business days
- Coordinated disclosure once a fix or mitigation is available

## Disclosure Policy

FluxPrompt follows coordinated disclosure. After a fix ships, maintainers may publish release notes or a changelog entry describing the issue at an appropriate level of detail.

## Non-Security Issues

Use public GitHub issues for normal bugs, documentation gaps, and feature requests that do not create a security risk.
